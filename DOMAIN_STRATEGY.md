## 前提：ドメインは「2階建て」で考える

制作して収益化する立場なので、層を分けて考える。

1. **各お店のサイトのアドレス**（陽気軒・次のカフェ・美容室…）← 本メモの主題
2. **自分の制作事業の窓口**（営業・実績ページ）← 別途あってもよい

ホスティングは全案とも **GitHub Pages（無料・SSL無料）** 前提。
よって差が出るのは **ドメイン代と管理の手間だけ**。

## 実際の飲食店サイトの実態

- ちゃんと営業している店のサイトは、ほぼ例外なく **店ごとの独自ドメイン** を持つ。
- 形は定番化している：`店名.com` / `店名.jp`（例 `youkiken.com` / `youkiken.jp`）。
- `.com` と `.jp` が二大定番。飲食では `.com` が最多、「日本の店」感なら `.jp`。
- `〜.github.io/shop-homepage/youkiken/` のようなパス型URLは「無料お試し・サンプル」の見え方で、
  名刺や食べログに載せづらく本番の信頼感が弱い → **本番には不向き**。

## オーソドックスな3方式

| | 方式 | URL例 | 年間コスト/店 | 向き |
|---|---|---|---|---|
| **A** | 店ごとに独自ドメイン | `youkiken.com` | 約1,000〜2,000円 | ★本命・本番案件 |
| **B** | 親ドメイン＋サブドメイン | `youkiken.屋号.com` | 0円（親1本のみ） | 試作・低価格プラン |
| **C** | パス型のまま（無料） | `…github.io/shop-homepage/youkiken/` | 0円 | 制作中・デモ専用 |

- **A 独自ドメイン**：検索でも店として独立評価。店の資産になり、制作者が手を引いても店に残せる。**飲食店サイトの標準形**。
- **B サブドメイン**：親ドメイン1本を持ち店ごとに頭を変える。管理1本で増えても楽。ただし「店自身のもの」感は薄い。
- **C パス型**：完全無料だが本番不向き。**作成中プレビュー専用**と割り切る。

## 単価の相場（国内）

| 項目 | 初年度 | 2年目以降（更新） |
|---|---|---|
| `.com` 1本 | 1円〜1,000円（キャンペーン） | 約1,500〜2,000円/年 |
| `.jp` 1本 | 0円〜2,000円 | 約3,000〜3,500円/年 |
| サブドメイン | 0円 | 0円（親ドメイン代に含む） |
| GitHub Pages | 0円 | 0円 |

※ 更新時に高くなるのが初心者の罠。試算は更新価格 `.com`=2,000円/年で固く見る。

## 3年・店舗数別の総コスト試算（`.com`想定）

「初年度1,000円＋2・3年目2,000円＝1店あたり3年で5,000円」で計算。

| | 1店 | 5店 | 10店 |
|---|---|---|---|
| **A 店ごと独自ドメイン** | 5,000円 | 25,000円 | 50,000円 |
| **B サブドメイン（親1本のみ）** | 5,000円 | 5,000円 | 5,000円 |
| **C パス型（無料）** | 0円 | 0円 | 0円 |

- **Aは店数に比例**して増えるが、**この代金はお店が払う実費**（自分の持ち出しではない）。
  見積りに「ドメイン代 実費 年2,000円」と載せれば済む。
- **Bは自分の持ち出しが親1本だけ**で固定。10店でも年2,000円。安いが店の資産にならない。
- **Cはタダ**だが本番の信頼感が出ない。

→ 自分の財布で見ると **A≒C（ほぼ0）／Bは固定2,000円/年**。
　お金より「店に渡る価値」と「手間」で選ぶのが正解。

## 手間（管理の手数）

| | ドメイン管理本数（10店時） | 更新作業 | 店をやめた時 |
|---|---|---|---|
| **A** | 10本（店ごと） | 店ごとに年1回 | 店にそのまま残せる ◎ |
| **B** | 1本 | 年1回だけ ◎ | 自分の管理下のまま（移管が手間） |
| **C** | 0本 | なし ◎ | 消すだけ |

A の「10本管理」は、お名前.com 等の管理画面に全部ぶら下げ＋**自動更新ON**にすれば実務上は苦にならない。

## 納品フローの具体例（A案・陽気軒を例に）

今の構成のまま実現できる流れ：

1. 制作中：`…github.io/shop-homepage/youkiken/`（今のまま・無料）で店に確認してもらう
2. GO が出たら **`youkiken.com` を取得**（お名前.com 等、5分・年1,000円）
3. リポジトリに `youkiken/CNAME` を置き、中身を `youkiken.com` にする
4. ドメイン側DNSで GitHub Pages を指定（A レコード4本＋CNAME）
5. GitHub の設定で「Enforce HTTPS」にチェック → **無料SSL自動付与**
6. 1〜2時間で `https://youkiken.com` が本番公開。サーバー代はずっと0円

### ⚠ 設計上の注意

- 独自ドメイン（CNAME）は **1リポジトリ＝1ドメイン** が基本。
- 今は1リポジトリに複数店を同居させているため、A案を本格化するなら
  **店ごとにリポジトリを分ける**のが素直。横展開の設計で効いてくる重要ポイント。

## ドメイン取得サービスの定番

- お名前.com / エックスドメイン / ムームードメイン
- 料金は更新まで含めるとエックスドメインが最安級。

## 結論（推し）

- **本番はA（店ごと独自ドメイン）一択**。自分の持ち出しはほぼ0（ドメイン代は施主負担）。
- **C は「制作中プレビュー専用」**として今の構成をそのまま活用。
- **B は「月額制の格安お任せプラン」を作る時の選択肢**としてキープ。
